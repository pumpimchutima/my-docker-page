Docker เบื้องต้น: คู่มือฉบับรวบรัดสำหรับผู้เริ่มต้น

Docker เป็นแพลตฟอร์มโอเพนซอร์สที่ช่วยให้นักพัฒนาสามารถสร้าง จัดส่ง และรันแอปพลิเคชันในรูปแบบที่เรียกว่า คอนเทนเนอร์ (Containers) คอนเทนเนอร์เหล่านี้เปรียบเสมือนกล่องที่บรรจุทุกสิ่งทุกอย่างที่แอปพลิเคชันต้องการ ไม่ว่าจะเป็นโค้ด, รันไทม์, ไลบรารี, และการตั้งค่าระบบ ทำให้แอปพลิเคชันสามารถทำงานได้อย่างสอดคล้องกันไม่ว่าจะรันอยู่บนสภาพแวดล้อมใดก็ตาม

ทำไมต้องใช้ Docker?

  • ความสอดคล้อง: ช่วยให้แอปพลิเคชันทำงานได้เหมือนกันทุกที่ ไม่ว่าจะบนเครื่องนักพัฒนา, เซิร์ฟเวอร์ทดสอบ, หรือเซิร์ฟเวอร์จริง ลดปัญหา "มันทำงานบนเครื่องผมนะ!"
  • ความรวดเร็ว: คอนเทนเนอร์เริ่มต้นได้เร็วกว่าเครื่องเสมือน (Virtual Machines) มาก ทำให้การพัฒนาและปรับใช้แอปพลิเคชันเป็นไปอย่างรวดเร็ว
  • ประสิทธิภาพ: ใช้ทรัพยากรน้อยกว่าเครื่องเสมือน เนื่องจากคอนเทนเนอร์ใช้เคอร์เนลของระบบปฏิบัติการร่วมกัน
  • พกพาง่าย: สามารถย้ายคอนเทนเนอร์จากสภาพแวดล้อมหนึ่งไปยังอีกสภาพแวดล้อมหนึ่งได้อย่างง่ายดาย
  • การจัดการที่ง่ายขึ้น: ช่วยให้การจัดการแอปพลิเคชันและบริการต่างๆ เป็นไปอย่างมีระเบียบและง่ายดายขึ้น โดยเฉพาะเมื่อทำงานกับแอปพลิเคชันที่มีส่วนประกอบหลายอย่าง (Microservices)

แนวคิดพื้นฐานของ Docker

  • Image (อิมเมจ): เป็นเหมือนแม่แบบ (Template) ที่อ่านอย่างเดียว (Read-only) สำหรับสร้างคอนเทนเนอร์ อิมเมจประกอบด้วยโค้ด แอปพลิเคชัน ไลบรารี การตั้งค่า และส่วนประกอบอื่นๆ ที่จำเป็น
  • Container (คอนเทนเนอร์): คืออินสแตนซ์ที่ทำงานอยู่ (Running instance) ของอิมเมจ คุณสามารถสร้างคอนเทนเนอร์หลายตัวจากอิมเมจเดียวกันได้
  • Dockerfile: คือไฟล์ข้อความที่ประกอบด้วยคำสั่งสำหรับสร้าง Docker Image
  • Docker Hub: เป็นคลังเก็บอิมเมจสาธารณะที่ใหญ่ที่สุด คล้ายกับ GitHub สำหรับโค้ด คุณสามารถอัปโหลดและดาวน์โหลดอิมเมจได้ที่นี่

ตัวอย่างการใช้งาน Docker เบื้องต้น

สมมติว่าคุณต้องการรันเว็บเซิร์ฟเวอร์ Nginx ด้วย Docker:

1. ติดตั้ง Docker

ก่อนอื่น คุณต้องติดตั้ง Docker Desktop บนเครื่องของคุณ (มีให้สำหรับ Windows, macOS และ Linux)

2. ดึง Docker Image

เปิด Terminal/Command Prompt แล้วพิมพ์คำสั่ง:

docker pull nginx

คำสั่งนี้จะดาวน์โหลดอิมเมจ Nginx ล่าสุดจาก Docker Hub

3. รัน Docker Container

docker run --name my-nginx -p 80:80 -d nginx
  • docker run: คำสั่งสำหรับสร้างและรันคอนเทนเนอร์
  • --name my-nginx: กำหนดชื่อคอนเทนเนอร์เป็น my-nginx
  • -p 80:80: เชื่อมต่อพอร์ต 80 ของเครื่องโฮสต์เข้ากับพอร์ต 80 ของคอนเทนเนอร์ (ทำให้คุณสามารถเข้าถึง Nginx ได้จาก http://localhost)
  • -d: รันคอนเทนเนอร์ในโหมดเบื้องหลัง (Detached mode)
  • nginx: ระบุอิมเมจที่จะใช้

4. ตรวจสอบ

เปิดเว็บเบราว์เซอร์ของคุณแล้วไปที่ http://localhost คุณจะเห็นหน้าต้อนรับของ Nginx

5. หยุดและลบคอนเทนเนอร์

docker stop my-nginx
docker rm my-nginx
